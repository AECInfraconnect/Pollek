P1 follow-up + P2 — ไฟล์ที่ส่งและวิธีเสียบ

ทำต่อจาก P1 (probation + dek-auth) ให้ครบวง และ implement P2 (ArcSwap + แตก main.rs) — โค้ดทั้งหมดอ้างอิง schema จริงของ repo (dek-ipc, proxy AppState)

1ไฟล์ที่ส่งมาในรอบนี้

ไฟล์ → ปลายทางใน repoทำอะไรหมวด
dek-core/src/ipc_client.rsIPC health roundtrip จริง (HealthCheck→HEALTHY) แทน TCP-connect placeholderP1 done

+ ไฟล์จากรอบก่อน: dek-auth/ (crate), dek-core/src/probation.rs, INTEGRATION_p1.rs

2ลำดับการเสียบ

  1. วาง dek-auth/ ที่ crates/dek-auth/ และ probation.rs, ipc_client.rs, supervisor.rs ที่ crates/dek-core/src/
  2. deps: dek-core เพิ่ม futures (มีใช้อยู่แล้วผ่าน Framed), tokio-util (มีแล้ว); dek-mcp-proxy เพิ่ม arc-swap = "1" + dek-auth = { path="../dek-auth" }; dek-cli เพิ่ม dek-config, self_replace (ถ้ายังไม่มี)
  3. main.rs: ลบ probation block เดิม (sleep 15s) แล้วใช้ Supervisor::bootstrap().run() (หรือถ้ายังไม่อยากแตกทั้งไฟล์ ใช้ snippet ใน INTEGRATION_p1.rs เสียบ probation อย่างเดียวก่อนได้)
  4. proxy: สลับ AppState เป็น state.rs (ArcSwap) + แทน JWT block ด้วย snap.verifier.verify(); stdio-wrapper ใช้ verifier ตัวเดียวกัน → ลบ JWT ซ้ำ
  5. dek-cli: register subcommand Rollback → rollback::run()
  6. updater.rs (แก้เล็ก): ใส่ target_path (current_exe) ลง marker JSON เพิ่ม เพื่อให้ rollback แม่นที่สุด (มี fallback derive จาก .bak อยู่แล้ว)
  7. bundle/jwt_config: เพิ่ม field audience เพื่อให้ proxy บังคับ aud ได้ (dek-auth จะ warn ถ้าไม่ตั้ง)
  8. ติดตั้ง systemd units + systemctl daemon-reload (หรือ sc failure ฝั่ง Windows ตาม INTEGRATION_p1)

3ครบวงของ Health-Gated Rollback (สรุปกลไก)

เมื่อ updater stage binary ใหม่ → เขียน marker + ขอ restart → binary ใหม่ boot → Supervisor เห็น marker → เข้า probation: ตรวจ IPC HEALTHY + mTLS ถึง cloud + bundle parse ต้องผ่าน 3 ครั้งติดภายใน 60s

  • ผ่าน → commit (ลบ marker + .bak)
  • ไม่ผ่าน → self_replace กู้ .bak → exit(1) → systemd Restart=on-failure พา binary เก่ากลับมา
  • ขึ้นแต่ค้าง (ไม่ ping watchdog) → WatchdogSec kill+restart
  • crash-loop เกิน StartLimitBurst → unit failed → OnFailure=dek-rollback.service → dekctl rollback restore + start
2 จุดที่ต้องปรับให้ตรง repo ตอน compile: (1) supervisor.rs อ้าง module ipc_server / bundle_loop ที่ต้องสกัดจาก main.rs เดิม (logic ย้ายตรง ๆ ตามคอมเมนต์ // MOVE:) — ถ้ายังไม่พร้อมแตกทั้งไฟล์ ใช้ probation snippet เดี่ยวก่อนได้ (2) state.rs เรียก snap.router.route_and_evaluate(...) — ปรับชื่อ method ให้ตรงกับ PolicyRouter จริง

4ทำไมเลือกแบบนี้ (ย้ำเหตุผล)

  • Service manager เป็นเจ้าของ lifecycle ไม่ใช่ watchdog orphan ที่ตายแล้วตาข่ายหาย — reboot-safe เพราะ marker อยู่บนดิสก์
  • "unhealthy" → exit non-zero เพื่อให้กลไก failure ของ systemd/SCM ทำงาน (จับได้ทั้ง crash และ "ขึ้นแต่พัง" ผ่าน watchdog)
  • ArcSwap ตัด per-request clone ของ JWKS ออก — read เป็น Arc clone O(1), reload atomic ไม่มี lock contention บน hot path
  • Supervisor ทำให้ทดสอบแต่ละ task ได้ + shutdown รวมศูนย์ + main เหลือแค่ wiring

โค้ดทั้งหมดอ้างอิง schema จริง (dek-ipc IpcMessage/IpcRequest::HealthCheck→HealthStatus{status:"HEALTHY"}, proxy AppState) — เป็น draft พร้อมปรับ; ยังไม่ได้รัน cargo build ในสภาพ workspace จริง
